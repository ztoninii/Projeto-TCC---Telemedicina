##  Sobre o Projeto
Este repositório contém o código-fonte do produto prático desenvolvido para o meu Trabalho de Conclusão de Curso (TCC). O projeto é um **Portal Educativo e de Agendamento de Telemedicina** desenhado especificamente para a população de Jundiaí - SP, integrando o atendimento das redes pública (SUS) e privada.

O portal adota o padrão visual e de usabilidade governamental (inspirado no Gov.br), garantindo uma interface confiável, limpa e altamente intuitiva.

##  Objetivo Acadêmico
O foco principal deste projeto é combater a infoexclusão e promover o **letramento digital em saúde**, especialmente para a população idosa. 

A pandemia de COVID-19 acelerou a digitalização da medicina, mas criou uma barreira tecnológica para os pacientes mais vulneráveis. Este portal atua como uma ferramenta de **humanização digital**, oferecendo:
1. **Educação:** Um passo a passo claro, em linguagem simples, sobre como se preparar para uma teleconsulta.
2. **Acessibilidade:** Ferramentas nativas que adaptam a interface para pessoas com baixa visão ou dificuldades de leitura.
3. **Autonomia:** Um sistema de simulação de agendamento simplificado e inteligente.

##  Principais Funcionalidades
* **Barra de Acessibilidade Padrão WCAG:** Botões para aumento/diminuição de fonte e ativação do Modo de Alto Contraste.
* **Navegação Single Page (SPA):** Transição suave entre abas (Como Funciona, Agendamento e FAQ) via JavaScript, sem recarregar a página.
* **Formulário Dinâmico:** O sistema identifica se o usuário deseja atendimento via SUS ou Convênio Privado e adapta os campos solicitados (ex: Cartão SUS vs. Carteira do Plano).
* **FAQ Interativo:** Uma seção de perguntas frequentes em formato "sanfona" (accordion) para tirar as principais dúvidas de forma objetiva.
* **Responsividade:** Layout *Mobile First*, perfeitamente adaptado para leitura em celulares, tablets e desktops.

##  Tecnologias Utilizadas
Para garantir um código leve, semântico e de fácil manutenção, o projeto foi construído utilizando tecnologias Front-End nativas (Vanilla):

* **HTML5:** Estruturação semântica visando a acessibilidade para leitores de tela.
* **CSS3:** Estilização com variáveis (Custom Properties) e Media Queries para responsividade.
* **JavaScript (ES6):** Manipulação da DOM (Document Object Model) para controle das abas, acessibilidade e validações do formulário.
